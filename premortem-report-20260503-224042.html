PREMORTEM REPORT

FairwayPro — App iOS de coaching golf • 3 mai 2026 • 8 agents d'investigation

On est en novembre 2026. FairwayPro a echoue. L'app stagne a moins de 10 utilisateurs payants. Le developpement est au point mort. Voici pourquoi c'est arrive, et comment l'eviter.

Synthese

L'echec le plus probable : Pas de distribution

L'app se construit dans le vide. Aucune strategie de go-to-market n'existe. Les coaches de golf ne cherchent pas d'apps sur l'App Store — ils decouvrent des outils par le bouche-a-oreille, les associations PGA, et les collegues au club. Sans presence dans ces canaux, FairwayPro lance dans le neant. 8 utilisateurs apres 3 mois, tous des contacts personnels. Noa interprete le silence comme un probleme produit et construit plus de features au lieu de trouver des utilisateurs.

L'echec le plus dangereux : Cle API exposee

La cle Anthropic sk-ant-api03-... est en clair dans AICoachScreen.js:7 et CoachApp.js:58. Des qu'elle est sur l'App Store, n'importe qui peut extraire cette cle d'un .ipa en 2 minutes. Consequence : factures API de centaines de dollars/jour, puis suspension du compte. Toutes les features IA meurent d'un coup. Les coaches payant CHF 40-60/mois pour une app "IA" demandent des remboursements. C'est le seul scenario qui peut causer des pertes financieres directes et detruire la reputation du produit simultanement.

L'hypothese cachee : "Un bon produit se vend tout seul"

A travers les 8 scenarios d'echec, une croyance invisible emerge : que le travail de developpement est le travail de lancement. Noa investit 100% de son temps a construire des features et 0% a trouver des utilisateurs, valider le pricing, ou preparer la distribution. L'hypothese non questionnee est que si Session Mode est assez bon, les coaches viendront naturellement. L'histoire des startups dit exactement le contraire — la distribution tue plus de produits que la qualite.

Le plan revise

1. Securite d'abord (cette semaine) : Migrer AICoachScreen.js et CoachApp.js vers le proxy Supabase claude-proxy. Revoquer la cle actuelle. Mettre une alerte budget chez Anthropic. Temps : 1 jour.

2. Enterrer la branche main (cette semaine) : Decision formelle : main est morte. Archiver les fichiers utiles (DailyBriefing) comme reference, mais tout le travail MVP continue sur restore-working-version. Pas de merge, pas de dette.

3. Trouver 5 coaches beta avant de coder Session Mode : Contacter l'ASG (Association Suisse de Golf), poster dans les groupes WhatsApp de coaches, aller physiquement dans 2-3 clubs. Objectif : 5 coaches actifs sur TestFlight qui ne sont pas des amis. Si c'est impossible, le probleme de distribution est confirme avant d'investir 2 mois de dev.

4. Lancer avec un prix d'early adopter : CHF 19/mois au lancement (pas 40). Valider la willingness-to-pay avec de vrais utilisateurs. Monter le prix quand Session Mode + AI Summary sont stables et que 20+ coaches l'utilisent activement.

5. Session Mode : prototype minimal d'abord : V1 = timer + notes texte attaches a une session. Pas de video in-app (les coaches filment deja avec Camera.app). L'IA resume les notes. Valider l'usage reel avant d'investir dans la capture video complexe.

Checklist pre-lancement

  • Cle API : Zero secret dans le code client. Toutes les calls AI passent par claude-proxy. Alerte Anthropic a $50/mois.
  • App Store readiness : SubscribeScreen n'est PAS le premier ecran. L'app est browsable sans payer. Disclosure AI visible. Privacy policy a jour (capture video de mineurs potentiels).
  • IAP fonctionnel : RevenueCat installe, sandbox teste, receipt validation en place AVANT soumission App Store.
  • 5 coaches non-amis sur TestFlight : Si ce nombre n'est pas atteint en 3 semaines d'efforts, pivoter la strategie de distribution avant de continuer le dev.
  • Branche unique : main archivee. Un seul flux de travail. Pas de merge futur a redouter.

8 scenarios d'echec analyses

Chaque carte represente un agent d'investigation independant

1. Cle API exposee

CRITIQUE • Impact immediat
La cle sk-ant-api03-... est hardcodee en clair dans 2 fichiers. Un .ipa est un zip — extraction en 2 minutes. Post-lancement : abus massif, factures de centaines de $/jour, puis suspension Anthropic. Toutes les features IA tombent. Le proxy Supabase existe deja mais n'est pas utilise partout.
Hypothese sous-jacente
Parce que l'app est sur TestFlight (audience fermee), il n'y a pas d'urgence a securiser la cle avant le lancement public.
Signaux d'alerte
Usage API Anthropic qui croit plus vite que le nombre d'utilisateurs. Toute exposition du repo (fork public, log CI) declenche des scanners automatiques.

2. Aucune strategie de distribution

CRITIQUE • Probabilite la plus elevee
L'App Store recoit 40 impressions le premier mois. Les coaches ne cherchent pas d'apps — ils ecoutent leurs collegues. Aucun partenariat avec une association golf, aucune presence dans les forums coaching. 8 utilisateurs en 3 mois, tous du reseau personnel. Noa construit plus de features au lieu de chercher des utilisateurs.
Hypothese sous-jacente
Construire un bon produit et le publier sur l'App Store suffit pour que les coaches le decouvrent et l'adoptent.
Signaux d'alerte
Zero contact avec une association golf avant le lancement. Moins de 10 testeurs TestFlight qui ne sont pas des contacts personnels.

3. Session Mode ne tient pas sa promesse

ELEVE • Risque produit fondamental
Le pilier du MVP n'existe pas encore. La capture video + micro + notes simultanées en React Native est un defi technique majeur. Meme si ca marche, l'UX sur un driving range (soleil, gants, ecran qui se verrouille) rend l'app plus lente que Camera.app + Notes. Les coaches essaient une fois et reviennent a leurs outils gratuits.
Hypothese sous-jacente
Les coaches adopteront une interface multi-outils inconnue pendant une lecon active, quand une app native gratuite fait deja le job principal.
Signaux d'alerte
En beta, les coaches lancent Camera.app au lieu de Session Mode. Moins de 1 video uploadee/semaine par coach = la value prop est morte.

4. Prix trop eleve pour le marche

ELEVE • Conversion killer
CHF 40-60/mois est plus cher que Netflix + Spotify combines. A ce prix, les coaches attendent un produit fini, pas un MVP. Le taux de conversion trial-to-paid reste sous 4%. Les coaches disent "c'est utile" mais "pas a ce prix". Apres Apple 30%, Noa touche CHF 28-42 par abonne. Les avis negatifs sur le prix tuent la croissance organique.
Hypothese sous-jacente
Les coaches paieront un prix SaaS premium pour remplacer des workflows qu'ils gerent actuellement gratuitement.
Signaux d'alerte
Taux de conversion sous 10% apres 2 mois. Les churned users reviennent aux outils gratuits (pas a des concurrents).

5. IAP jamais integre

ELEVE • Bloquant revenue
Le SubscribeScreen existe mais AUCUNE librairie de paiement n'est installee. Pas de RevenueCat, pas d'expo-iap dans package.json. Le bouton "S'abonner" n'existe pas — juste "Se connecter". Noa grant manuellement l'acces via Supabase, masquant l'urgence. Apple rejette la soumission : prix affiches sans flow d'achat fonctionnel.
Hypothese sous-jacente
Le paywall peut etre construit UI-first et branche aux paiements "plus tard", sans realiser qu'Apple exige l'infra de paiement avant tout langage de monetisation.
Signaux d'alerte
package.json sans dependance IAP. SubscribeScreen sans bouton d'achat fonctionnel. MASTER_CONTEXT mentionne RevenueCat mais le code ne l'a pas.

6. Rejets App Store repetes

MOYEN • Delai de lancement
Premier rejet : 3.1.1 (SubscribeScreen = premier ecran, pas de navigation possible sans payer). Deuxieme rejet : cle API en clair + pas de disclosure AI. Chaque rejet = 1-2 semaines de fix + resoumission. 3 mois de fenetre de lancement perdus. Les early adopters TestFlight churnent en attendant.
Hypothese sous-jacente
Passer la review TestFlight (qui n'a pas de scrutin IAP/paywall) equivaut a etre pret pour l'App Store.
Signaux d'alerte
Le commit recent "fix: remove external URL button for App Store 3.1.1 compliance" montre qu'Apple a deja flag un probleme — le symptome a ete traite mais pas la cause racine.

7. Divergence des branches

MOYEN • Dette technique
main a ajoute i18n, ThemeContext, expo-haptics, expo-linear-gradient — chaque ecran importe useTheme() et useTranslation(). La branche stable n'a rien de tout ca. Merger n'importe quel fichier de main casse instantanement. DailyBriefing (feature MVP) est sur main dans une architecture incompatible. Des semaines de reconciliation au lieu de dev.
Hypothese sous-jacente
Le travail sur main peut etre cherry-pick dans la branche stable plus tard, alors que la divergence est architecturale (dependances, theme, i18n).
Signaux d'alerte
main reste au commit "WIP: backup before debugging crash" pendant 30+ jours. Toute feature MVP qui touche un fichier existant aussi sur main.

8. Goulot solo developer

MOYEN • Risque structurel
Noa fait tout seul : dev, design, App Store, marketing, support, bugs. Session Mode (la feature la plus complexe) n'est pas construite. Un evenement de vie (burnout, job, perso) = arret total du projet. La frequence de commits passe de hebdomadaire a mensuelle. Les testeurs TestFlight se desengagent. Le projet entre en mode "j'y reviendrai".
Hypothese sous-jacente
Noa aura une capacite de developpement consistante et ininterrompue pendant 6 mois.
Signaux d'alerte
Moins d'un commit par semaine pendant 2 semaines consecutives. Taux de reponse des testeurs TestFlight sous 30%.
